### 方式 2️⃣：免费部署到 GitHub Pages（推荐！）

**适合**：公网访问，所有人都能看到，完全免费

#### 🎯 分 5 步完成：

**Step 1: 创建 GitHub 仓库**
- 访问 https://github.com/new
- 仓库名：`objection-assistant`
- 选择 Public → Create

**Step 2: 上传文件**

使用 Git 命令：
```bash
cd /Users/bobo/WorkBuddy/20260331155330
git init
git add .
git commit -m "Initial commit"
git remote add origin https://github.com/<你的用户名>/objection-assistant.git
git branch -M main
git push -u origin main
```

**Step 3: 启用 GitHub Pages**
- 进入仓库 Settings → Pages
- Branch 选择 `main`
- Folder 选择 `/ (root)`
- Save

**Step 4: 等待生效**
- 等待 2-3 分钟

**Step 5: 访问你的网站**
```
https://<你的用户名>.github.io/objection-assistant
```

**✨ 完成！**

**优点：** 
- ✅ 完全免费
- ✅ 无限使用
- ✅ 任何人都能访问
- ✅ 支持手机/电脑

**缺点：** 
- 静态网站，高级功能有限制

### 方式 3️⃣：部署到云服务器（企业级）

**适合**：大团队，需要高可用性，有预算

#### 推荐平台（都有免费试用）：

| 平台 | 成本 | 优点 | 部署时间 |
|------|------|------|--------|
| **Render** | 免费 | 简单，支持 Python | 5 分钟 |
| **Railway** | $5/月 | 性能好，易用 | 5 分钟 |
| **Vercel** | 免费 | 部署快，自动化 | 3 分钟 |
| **阿里云** | ¥9.9/月 | 国内快，售后好 | 10 分钟 |

**快速部署到 Render：**

1. 访问 https://render.com
2. Sign up 并关联 GitHub
3. New → Web Service
4. 选择 `objection-assistant` 仓库
5. 配置：
   - Build Command: `pip install -r requirements.txt`
   - Start Command: `python web_app.py`
6. Deploy

✅ 完成！获得公网 URL

## ❓ 常见问题

**Q: 部署后访问不了？**
A: 检查是否按照步骤完整操作。常见原因：
   - GitHub Pages 需要等 2-3 分钟生效
   - 检查仓库是否为 Public
   - 检查 index.html 是否上传

**Q: 可以在手机上访问吗？**
A: 完全可以！使用方式 2 后，复制 URL 到手机浏览器打开即可。

**Q: 我没有 GitHub 账户**
A: 免费注册一个（30 秒搞定）：https://github.com/signup

**Q: 为什么页面加载很慢？**
A: 
   - GitHub Pages 可能在地理位置较远的地方较慢
   - 使用方式 3（云服务器）可以选择国内服务商加快速度

**Q: 可以加密访问吗？**
A: GitHub Pages 免费提供 HTTPS。云服务器也都支持 HTTPS。

**Q: 如何统计访问人数？**
A: 可以使用 Google Analytics 或其他分析工具。
