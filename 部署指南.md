## 公网部署（推荐）

### 方式一：使用 Render（完全免费，无需信用卡）

#### Step 1: 准备代码

在项目中创建 `Procfile` 文件：

```bash
cat > Procfile << 'EOF'
web: python web_app.py
EOF
```

创建 `runtime.txt`：

```bash
cat > runtime.txt << 'EOF'
python-3.9.6
EOF
```

更新 `requirements.txt`：

```bash
pip freeze > requirements.txt
```

#### Step 2: 上传到 GitHub

```bash
# 初始化 Git 仓库
git init
git add .
git commit -m "Initial commit: 异议查询助手在线版"

# 推送到 GitHub（需要先在 GitHub 创建仓库）
git remote add origin https://github.com/你的用户名/objection-assistant.git
git branch -M main
git push -u origin main
```

#### Step 3: 部署到 Render

1. 访问 https://render.com
2. 点击 "Get Started"
3. 选择 "New +" → "Web Service"
4. 连接 GitHub 账户，选择仓库
5. 配置：
   - **Name**: `objection-assistant`
   - **Runtime**: `Python 3`
   - **Build Command**: `pip install -r requirements.txt`
   - **Start Command**: `python web_app.py`
   - **Instance Type**: `Free`
6. 点击 "Create Web Service"

**✨ 完成！** 你将获得一个公网 URL，如：
```
https://objection-assistant.onrender.com
```

## 企业级部署

### 使用 Docker + AWS EC2 / 阿里云 ECS

#### Step 1: 创建 Dockerfile

```dockerfile
FROM python:3.9-slim

WORKDIR /app

COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt

COPY . .

ENV FLASK_APP=web_app.py

EXPOSE 5000

CMD ["python", "web_app.py"]
```

#### Step 2: 构建镜像

```bash
docker build -t objection-assistant .
```

#### Step 3: 运行容器

```bash
docker run -p 5000:5000 objection-assistant
```

#### Step 4: 部署到云服务器

**AWS EC2:**
```bash
# 登录 EC2 实例
ssh -i your-key.pem ec2-user@your-instance-ip

# 安装 Docker
sudo yum install docker -y
sudo systemctl start docker

# 上传 Docker 镜像并运行
docker run -p 80:5000 -d objection-assistant
```

**阿里云 ECS:**
```bash
# 推送到 Container Registry
docker tag objection-assistant registry.aliyuncs.com/your-namespace/objection-assistant
docker push registry.aliyuncs.com/your-namespace/objection-assistant

# 在 ECS 中运行
docker run -p 80:5000 -d registry.aliyuncs.com/your-namespace/objection-assistant
```

### 使用 Nginx 反向代理

```nginx
server {
    listen 80;
    server_name objection.yourdomain.com;

    location / {
        proxy_pass http://127.0.0.1:5000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```

### 使用 Supervisor 进程管理

```ini
# /etc/supervisor/conf.d/objection.conf
[program:objection]
directory=/home/user/objection-assistant
command=python web_app.py
autostart=true
autorestart=true
stderr_logfile=/var/log/objection.err.log
stdout_logfile=/var/log/objection.out.log
```

## 常见问题

### Q1: 如何在 URL 中隐藏端口？

**A:** 使用 Nginx 反向代理：

```nginx
server {
    listen 80;
    server_name objection.yourcompany.com;

    location / {
        proxy_pass http://127.0.0.1:5000;
    }
}
```

然后访问 `http://objection.yourcompany.com`

### Q2: 如何添加用户认证？

**A:** 修改 `web_app.py`，添加以下代码：

```python
from flask_httpauth import HTTPBasicAuth
from werkzeug.security import check_password_hash

auth = HTTPBasicAuth()
users = {
    "admin": "password123",  # 用户名:密码
}

@auth.verify_password
def verify_password(username, password):
    if username in users:
        return users.get(username) == password

@app.route('/')
@auth.login_required
def index():
    return render_template_string(HTML_TEMPLATE)
```

### Q3: 如何加速访问？

**A:** 使用 CDN（如 Cloudflare）：

1. 访问 https://www.cloudflare.com
2. 添加你的域名
3. 将 DNS 指向 Cloudflare
4. 启用缓存和 CDN

### Q4: 如何处理高并发？

**A:** 使用 Gunicorn + Nginx：

```bash
# 安装 Gunicorn
pip install gunicorn

# 使用 4 个 worker 进程启动
gunicorn -w 4 -b 0.0.0.0:5000 web_app:app
```

### Q5: 更新异议数据后需要重启吗？

**A:** 

- **开发模式**（`python web_app.py`）：自动重载，无需重启
- **生产模式**（Gunicorn）：需要重启进程

```bash
# 重启 Gunicorn
pkill -f gunicorn
gunicorn -w 4 -b 0.0.0.0:5000 web_app:app
```

### Q6: 可以在手机上访问吗？

**A:** 完全可以！

- 同一 WiFi：访问 `http://<电脑IP>:5000`
- 不同网络：部署到公网后即可访问
